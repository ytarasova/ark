Contributing

Development setup, project structure, testing patterns, and code style.

Prerequisites

  • Bun -- runtime and package manager (required, Node.js will not work)
  • tmux -- session multiplexer for running agents
  • git -- for worktree-based session isolation
  • gh (optional) -- GitHub CLI for auto-PR creation
Bun-only. Ark uses bun:sqlite, Bun.serve(), Bun.sleep(), and Bun FFI. It will not run under Node.js.

Setup

$ git clone https://github.com/ytarasova/ark.git
$ cd ark
$ make install   # bun install + symlink ark to /usr/local/bin

Development Workflow

# Hot-reload: TypeScript watch + web dev server
$ make dev

# Run the TUI from source
$ make tui

# Launch just the web dev server (needs separate ark web on :8420)
$ make dev-web

# Launch the Electron desktop app
$ make desktop

Module Layers

From bottom to top:

  1. packages/types/ -- Domain interfaces. No logic, no dependencies.
  2. database.ts / database-sqlite.ts / database-postgres.ts -- IDatabase abstraction.
  3. repositories/ -- SQL CRUD behind typed classes. Column whitelists prevent injection.
  4. stores/ -- Resource stores with three-tier file resolution.
  5. services/session.ts -- Lifecycle facade: start, stop, resume, complete, pause, delete.
  6. services/session-orchestration.ts -- All orchestration (dispatch, advance, fork, fan-out).
  7. app.ts -- AppContext wires everything together. Boot/shutdown lifecycle.
  8. conductor.ts -- HTTP server (:19100), hook status, channel relay.

Key Entry Points

Entry PointAccess Pattern
AppContextRepos: app.sessions, app.computes, app.events. Stores: app.flows, app.skills, app.agents. Services: app.sessionService.
SessionServiceSimple lifecycle. Delegates complex ops to session-orchestration.ts.
session-orchestration.tsEvery function takes app: AppContext as first argument. No getApp() calls.

Testing

Tests use bun:test. Always use make targets -- never call bun test directly.

# Run all tests (sequential, builds deps first)
$ make test

# Run a single test file
$ make test-file F=packages/core/__tests__/session.test.ts

# Run E2E tests (TUI + Web)
$ make test-e2e

# Watch mode
$ make test-watch
Never run tests in parallel. Tests share ports (19100, 19200, 19300), globalThis state, and SQLite databases. Bun runs test files concurrently by default, which causes port collisions and phantom failures. The Makefile enforces --concurrency 1.

Test Isolation Pattern

Use AppContext.forTest() for isolated test contexts:

import { AppContext, setApp, clearApp } from "../app.js";

let app: AppContext;
beforeAll(async () => {
  app = AppContext.forTest();
  await app.boot();
  setApp(app);
});
afterAll(async () => {
  await app?.shutdown();
  clearApp();
});

AppContext.forTest() creates a temporary directory with an isolated database. It skips the conductor, metrics, and signal handlers. The temp directory is cleaned up on shutdown.

withTestContext() Helper

A convenience wrapper that manages the beforeEach/afterAll lifecycle automatically:

import { withTestContext } from "./test-helpers.js";
const { getCtx } = withTestContext();

it("creates a session", () => {
  const app = getCtx();
  // AppContext is isolated, booted, and ready
  app.sessions.create({ ... });
});

waitFor() Utility

Polls a condition until it returns true (or times out). Useful for testing async state transitions:

await waitFor(() => getSession(id).status === "running");

E2E Tests

E2E tests require the dist/ directory to be built first:

# Build first, then run E2E
$ make dev  # or just tsc
$ make test-e2e

TUI E2E tests use a browser harness (xterm.js + real pty + real tmux). Web E2E tests use Playwright.

Code Style

  • TypeScript with strict: false (implicit any allowed, no strict null checks)
  • ES modules ("type": "module") -- always use .js import extensions even in TypeScript
  • React + Ink for TUI components, React + Vite + Tailwind for Web
  • YAML for agent/flow/runtime/recipe definitions
  • SQLite for local, PostgreSQL for hosted mode (via IDatabase abstraction, no ORM)
  • Never use em dashes (U+2014) -- use hyphens (-) or double dashes (--) instead

Import Extensions

All relative imports must use .js extensions, even in TypeScript files:

// Correct
import { foo } from "./bar.js";

// Wrong -- breaks at runtime
import { foo } from "./bar";

No Global State

Pass all state as arguments. Utility functions should never call getApp() or ARK_DIR(). Every orchestration function takes app: AppContext as its first argument.

Schema Changes

Ark does not have formal migrations (pre-pilot, no production data worth preserving). The authoritative schema lives in packages/core/repositories/schema.ts.

  • Adding a new table or column with DEFAULT: transparent, tables recreate with IF NOT EXISTS.
  • Renaming or changing a column type: delete the database and let it recreate: rm ~/.ark/ark.db

$ make self

$ make self-quick
