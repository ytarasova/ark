Web Dashboard

Browser-based UI and native desktop app for managing Ark sessions.

Starting the Web Dashboard

Run ark web to start the HTTP server. By default it listens on port 8420.

$ ark web                           # http://localhost:8420
$ ark web --port 9000               # custom port
$ ark web --read-only               # no mutations allowed
$ ark web --token my-secret-token   # require Bearer token auth
OptionDefaultDescription
--port8420Port for the web server
--read-only--Disable all mutation endpoints
--token--Require Authorization: Bearer <token> on every request

Views

All views are accessible from the sidebar. Each view mirrors the equivalent tab in the TUI.

Session Interactions

The Sessions view provides the full session lifecycle from the browser. Available actions from the session list and detail panel:

  • Create -- New Session modal with repo, flow, summary, compute, and group fields
  • Dispatch / Stop / Restart -- lifecycle controls available from the detail panel
  • Send messages -- inline chat input for talking to a running agent via the MCP channel
  • Interrupt -- sends Ctrl+C to the agent without killing the tmux session
  • Pause / Advance / Complete -- gate controls for manual-gate flow stages
  • Fork -- decompose the session into parallel child sessions
  • Archive / Restore -- hide completed sessions without deleting them

Diff Preview and PR Creation

The session detail panel includes a diff preview section that shows a git diff --stat of the worktree branch. From this panel you can:

  • Browse the list of changed files and track which ones you have reviewed
  • See files modified since your last review, flagged with a change indicator
  • Create a GitHub PR via gh pr create directly from the browser
  • Merge the worktree branch into the base branch
Note: PR creation requires gh (GitHub CLI) to be installed and authenticated on the machine running ark web.

Cost Tracking

The Costs view provides a breakdown of token usage across all sessions:

  • Per-session token breakdown: input tokens, output tokens, cache reads, and cache writes
  • Cost per session calculated from per-model pricing
  • Aggregate totals across all sessions and date ranges
  • Budget usage shown against daily, weekly, and monthly limits configured in settings

Memory / Knowledge

The Memory view manages the knowledge graph -- memories, learnings, and knowledge stats. Agents access this at dispatch time:

  • Add new memories with optional tags, scope, and importance level
  • Search memories by keyword
  • Delete individual memories or clear all memories within a scope

Live Updates

The web dashboard uses Server-Sent Events (SSE) for real-time updates. There is no polling -- the server pushes status changes, new events, and agent progress directly to the browser as they happen. Session status badges, event logs, and agent output all update automatically.

Access Control

Two access control mechanisms are available for deployments that need protection:

Token Authentication

Start the server with --token <secret>. All requests must include the header:

Authorization: Bearer <your-token>

Requests without a valid token receive a 401 Unauthorized response. The token is set once at startup and does not rotate.

Read-Only Mode

Start with --read-only to disable all mutation endpoints. GET requests and SSE streams remain available. Any POST, PUT, PATCH, or DELETE request returns 403 Forbidden. Useful for sharing a dashboard with observers who should not be able to control sessions.

Warning: ark web binds to 0.0.0.0 by default, making it reachable on all network interfaces. Use --token or a firewall rule when running on a shared or remote machine.

Desktop App

Ark includes an Electron app that wraps the web UI in a native window. It provides a dedicated macOS/Windows/Linux application without requiring a separate browser tab.

Running the Desktop App

$ make desktop

This launches the Electron app in development mode. On startup it boots ark web as a child process on a free port, then opens a BrowserWindow pointing at that port. Closing the window shuts down the web server.

Prerequisites

Before running for the first time, install Electron and electron-builder:

$ make desktop-install

The desktop app also requires Bun, Git, and ark to be available on PATH.

Building for Distribution

$ make desktop-build

Packages the app for distribution using electron-builder. Output is written to packages/desktop/out/:

PlatformOutput format
macOS.dmg and .zip
Windows.exe (NSIS installer)
Linux.AppImage and .deb

Native Features

  • macOS traffic light buttons (close, minimize, zoom) with native window chrome
  • Native menu bar with standard application menus
  • External links open in the system browser rather than inside the app window
  • Dark background matches the web dashboard theme

The desktop app and the web dashboard share the same codebase. Any feature available at http://localhost:8420 is also available in the desktop app.
