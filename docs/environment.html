Port Assignments

ServicePortNotes
Conductor19100Hardcoded. HTTP server for hook status and channel relay.

ArkD19300Default. Universal agent daemon on every compute target.
LLM Router8430Default. OpenAI-compatible proxy.
TensorZero3000Default. Optional LLM gateway behind the router.
Web UI8420Default. Ark web dashboard (Vite).
Vite HMR5173Dev only. Vite dev server with hot module replacement.
Port collisions: Tests share ports 19100, 19200, and 19300. This is why tests must always run sequentially via make test -- never in parallel.

Three-Tier Resource Resolution

Agents, flows, skills, recipes, and runtimes all use three-tier resolution. When Ark looks up a resource by name, it checks these locations in order (highest priority first):

  1. Project: .ark/<type>/<name>.yaml (or .md for skills) in the current repo
  2. Global: ~/.ark/<type>/<name>.yaml (user-level)
  3. Builtin: <type>/<name>.yaml shipped with Ark

Project-level definitions override global ones, which override builtins. This lets you customize any resource per-repo without modifying Ark itself.
