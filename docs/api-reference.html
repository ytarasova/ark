API Reference

JSON-RPC 2.0 protocol, ArkClient, and remote mode.

Overview

Ark exposes a JSON-RPC 2.0 API for all operations. The same API powers the CLI, Web dashboard, Desktop app, and remote clients. The protocol version is 0.8.0.

Source: packages/server/ (handlers), packages/protocol/ (client).

Transport Layers

TransportUse CaseEncoding
StdioLocal CLIJSONL (JSON + newline)
WebSocketRemote clients, Web dashboardJSON-RPC over WS (optional Bearer token auth)

Remote Mode

Connect to a remote Ark server instead of running locally:

# Via CLI flags
$ ark --server https://ark.company.com --token ark_default_xxx session list

# Via environment variables
$ export ARK_SERVER=https://ark.company.com
$ export ARK_TOKEN=ark_default_xxx
$ ark session list

# Web UI in remote mode (proxy)
$ ark web --server https://ark.company.com --token ark_default_xxx

When remote mode is active, the CLI creates a WebSocket ArkClient to the remote server instead of booting a local AppContext.

ArkClient

The typed JSON-RPC client that wraps the transport layer. All UI surfaces use ArkClient for communication.

Initialization

import { ArkClient, createWebSocketTransport } from "@ark/protocol";

const { transport, ready } = createWebSocketTransport(
  "wss://ark.company.com",
  { token: "ark_default_xxx" }
);
await ready;

const client = new ArkClient(transport);
await client.initialize({ subscribe: ["session/*"] });

Subscriptions

Subscribe to real-time notifications using patterns:

  • "**" -- all notifications
  • "session/*" -- all session-related notifications
  • "session/updated" -- exact method match
client.on("session/updated", (data) => {
  console.log("Session changed:", data.sessionId);
});

-32601METHOD_NOT_FOUNDNo handler for the requested method
-32602INVALID_PARAMSMissing or invalid parameters
-32603INTERNAL_ERRORUnexpected server error

-32002SESSION_NOT_FOUNDSession ID does not exist
-32003EXECUTOR_ERRORAgent executor failed (launch, kill, etc.)
-32004GATE_NOT_PENDINGTried to approve a gate that is not pending
-32005OVERLOADEDServer at capacity (too many concurrent sessions)

OpenRPC Schema

A machine-readable OpenRPC 1.3 schema is available with full request/response JSON Schemas for every method. Use it with the OpenRPC playground, mock servers, or client code generators. Download: openrpc.json.

Regenerate after adding new Zod schemas: bun run scripts/generate-openrpc.ts

Authentication

In hosted mode, API keys authenticate requests. Key format: ark_<tenantId>_<secret>.

Keys are managed via:

# Create a new API key for a tenant
$ ark api-key create --tenant default --role admin

# List keys
$ ark api-key list

# Revoke a key
$ ark api-key revoke <key-id>

Roles: admin (full access), member (session management), viewer (read-only).

Parameter Validation

All RPC handlers validate parameters at the boundary using extract<T>() from packages/server/validate.ts. Missing required parameters return INVALID_PARAMS (-32602).
