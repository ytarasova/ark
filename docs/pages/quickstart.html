Quick start

Install Ark, launch your first agent session, and watch a DAG flow complete — in about two minutes.

Prerequisites

Ark is Bun-only and uses tmux to multiplex agent processes. Before you install:

  • Bun ≥ 1.1 — curl -fsSL bun.sh/install | bash
  • tmux ≥ 3.2 — brew install tmux or your package manager
  • At least one agent runtime: Claude Code, Codex, Gemini CLI, or Goose
i
Ark won't fall back if tmux is missing — check tmux -V before installing.

Install

The one-liner links ark into /usr/local/bin and runs bun install:

bash
$ curl -fsSL ark.run/install | bash
› downloading ark-v0.9.1-darwin-arm64…
› bun install (147 packages)
› linked ark → /usr/local/bin/ark
› ~/.ark/ initialized

installed. try: ark --help

Or, from source:

bash
$ git clone https://github.com/ark-run/ark && cd ark
$ make install

Your first session

A session is one agent working against one repo on one compute target. Kick one off inline:

bash
$ cd ~/projects/my-app
$ ark new "explain the auth middleware" --runtime claude

  ◆ session   ses_01hnr8x9m4
  ◇ compute   local
  ◇ runtime   claude-code

attaching… (ctrl-b d to detach)

Ark boots arkd inside a tmux window, spawns the runtime, and streams output to your terminal. Detach whenever — the session keeps running.

✦
Use ark ls to list running sessions, ark attach <id> to re-attach, ark stop <id> to terminate.

Run a flow

Flows are DAGs of agent tasks. The built-in autonomous-sdlc flow plans, specs, implements, tests, and prepares a PR:

bash
$ ark run autonomous-sdlc "add rate limiting to /api/messages" \
    --compute ec2-warm \
    --router balanced

  ◆ flow  autonomous-sdlc (v3)
  ├─ ✓ plan          4 subtasks
  ├─ ✓ spec          api contract v2
  ├─ ▸ implement     editing middleware/rate.ts
  └─ ◌ test + ship

Each node runs in its own session, with shared context via the knowledge graph. You can define your own flows in YAML.

Open the dashboard

The web dashboard gives you a live view of sessions, flows, and agent transcripts:

bash
$ ark dev
› api    :8420
› web    :5173  (Vite HMR)
› daemon :19400 (auto-started)

open http://localhost:5173
!
On first run, the dashboard asks which theme + workspace to use. Your pick persists in ~/.ark/config.yaml.
