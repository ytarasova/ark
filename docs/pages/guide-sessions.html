Sessions

Everything you can do with a running agent — spawn, attach, pause, branch, snapshot, stop.

Spawn a session

$ ark new "refactor auth middleware" \
    --runtime claude \
    --compute ec2-warm \
    --branch feature/auth-refactor

Attach & detach

ark attach <id> joins the tmux window; Ctrl-B D detaches without killing it. The session keeps running.

Event stream

Tail the session's event stream as newline-delimited JSON:

$ ark tail ses_01hnr8x9m4 --format json
{"t":"tool.call","tool":"str_replace","path":"auth.ts"}
{"t":"tool.result","ok":true}
{"t":"message","role":"assistant","content":"edited auth.ts"}

Branch from a checkpoint

Every tool call creates an implicit checkpoint. Fork a session from any checkpoint to explore an alternative:

$ ark branch ses_01hnr8x9m4@12 "try PostgreSQL instead"
  ◆ branched: ses_01hnr8x9m4 → ses_01hnr91k2a (at event #12)

Stop & clean up

$ ark stop ses_01hnr8x9m4           # graceful
$ ark stop ses_01hnr8x9m4 --force   # SIGKILL the whole tmux window
$ ark gc                             # prune stopped sessions older than 7d
!
On ec2-* providers, stopping a session releases the VM. Use ark pause to keep the VM warm for 10 minutes.
