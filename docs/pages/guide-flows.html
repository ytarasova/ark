Flows & DAGs

Compose multi-agent pipelines as YAML. Branch, retry, gate on human review — all declarative.

Anatomy of a flow

A flow is a named DAG. Each node specifies an agent, a runtime, optional dependencies, and optional gates:

name: triage
version: 1

inputs:
  issue_url: { type: string, required: true }

nodes:
  classify:
    agent: issue-triager
    runtime: claude
    inputs:
      url: "{{inputs.issue_url}}"

  reproduce:
    agent: repro-builder
    runtime: codex
    needs: [classify]
    // only run if classify tagged it a bug
    when: "{{classify.tags | includes('bug')}}"

  fix:
    agent: fixer
    runtime: claude
    needs: [reproduce]
    gate: human-review

Inputs & outputs

Flows take typed inputs and expose each node's output as {{<node>.<field>}}. Templating uses Nunjucks.

Human gates

A node with gate: human-review pauses the flow after it completes and waits for approval in the dashboard or via ark approve <flow>#<node>.

Fan-out, fan-in

nodes:
  plan: { agent: planner, runtime: claude }

  implement:
    agent: coder
    runtime: claude
    needs: [plan]
    for_each: "{{plan.subtasks}}"   // fan-out

  merge:
    agent: merger
    runtime: claude
    needs: [implement]            // fan-in

Retry & recovery

Every node can specify retries: N and an optional on_reject prompt that's templated with failure context and fed back to the agent.
