Compute targets

Ark dispatches sessions as a (compute, runtime) pair. Both axes are pluggable — same ComputeTarget interface, different substrate.

Providers at a glance

localinstantday-to-day work

Pick a target per run

$ ark new "explore" --compute local
$ ark run flow    --compute ec2-warm
$ ark run flow    --compute k8s,runtime=devcontainer

Warm pools

Configure ec2-warm to keep N instances pre-booted. Pool state is cached in ~/.ark/compute-pool.db.

# ~/.ark/config.yaml
compute:
  ec2-warm:
    region: us-west-2
    instance_type: c7g.xlarge
    pool_size: 4
    idle_ttl: 10m

BYO provider

Drop a TypeScript module in packages/compute/adapters/ implementing Compute + Runtime. It's dispatched by (compute_kind, runtime_kind) — no core changes needed.
