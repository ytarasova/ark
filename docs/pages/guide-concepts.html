Core concepts

Four primitives power everything in Ark: sessions, flows, agents, and compute targets. Learn what each does and how they compose.

Session

A session is one agent process working against one repo on one compute target. It has a unique id, a tmux window, an arkd handle, and an event stream you can tail.

lifecycle
provisioning → booting → running → (paused)? → completed | failed | stopped

Sessions are cheap to spawn and persist beyond your shell. Your terminal attaches and detaches; the session runs in the background until it exits or you stop it.

Flow

A flow is a DAG of nodes, each of which boots its own session. The built-in autonomous-sdlc flow has five nodes: plan, spec, implement, test, ship. Edges declare dependencies and optional review gates.

flows/autonomous-sdlc.yaml
name: autonomous-sdlc
version: 3

nodes:
  plan:      { agent: planner,  runtime: claude }
  spec:      { agent: speccer,  runtime: claude,  needs: [plan] }
  implement: { agent: coder,    runtime: claude,  needs: [spec] }
  test:      { agent: tester,   runtime: codex,   needs: [implement] }
  ship:      { agent: releaser, runtime: claude,  needs: [test],
              gate: human-review }

Agent

An agent is a YAML persona — system prompt, tool permissions, output contract. Agents are runtime-agnostic: the same coder agent runs under Claude Code today and Gemini CLI tomorrow.

See Flows & DAGs for how nodes pick an agent + runtime pair.
