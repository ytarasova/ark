LLM router

OpenAI-compatible proxy with three built-in policies and per-provider circuit breakers. Route any agent's calls by cost, latency, or model affinity.

How it works

Agent runtimes point at http://localhost:8420/v1 instead of a model provider. The router picks a backend per request based on the active policy, then transparently proxies the response.

i
The router is enabled by default in ark dev. Disable with --no-router.

Policies

PolicyChooses by
cheapestlowest $/1k tokens that satisfies the model class
fastestlowest observed p50 latency in the last 5 min
balancedweighted mix of cost, latency, and error rate

Configure

# ~/.ark/config.yaml
router:
  policy: balanced
  backends:
    - name: anthropic
      model_class: [sonnet, haiku]
      weight: 1.0
    - name: openai
      model_class: [gpt-4, gpt-4-mini]
      weight: 0.6
    - name: self-hosted
      url: http://gpu-box:8000
      model_class: [llama-3]

Circuit breakers

Each backend has a breaker that trips after 5 consecutive 5xx errors or a p99 latency > 30s for 60 seconds. Tripped backends cool down for 2 minutes, then retry one request to recover.

Inspect

$ ark router stats
  backend       p50     p99    errs  state
  anthropic     780ms   4.2s    0%   ● up
  openai        620ms   3.1s    2%   ● up
  self-hosted   1.9s   12.4s   18%   ○ half-open
