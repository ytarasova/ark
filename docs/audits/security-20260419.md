# Security audit - 2026-04-19

Defensive-security review of the Ark codebase at commit `db2b593`. All
confirmed findings are fixed in this PR; triaged-as-safe items are listed at
the bottom for transparency.

Severity breakdown:

- Critical: 4 (all fixed)
- High: 4 (all fixed)
- Medium: 3 (2 fixed, 1 is a dev-only transitive CVE - notes below)
- Low / informational: 0

`bun audit` reports 2 moderate transitive CVEs (`yaml` via `vite`,
`brace-expansion` via `eslint` / `@typescript-eslint`). Both are dev-only
dependencies and are not shipped in the Ark runtime; no runtime upgrade is
required in this PR.

## Threat model per package

### `packages/server` (JSON-RPC handlers)

- **Attacker**: a remote caller holding a valid API key for tenant A who
  tries to exercise handlers that either (a) bypass tenant scoping, (b)
  accept privileged server-local arguments (paths, pids, container names),
  or (c) are missing from the `WRITE_METHODS` set and therefore bypass the
  read-only / viewer-role check in `packages/core/hosted/web.ts`.
- **Asset**: other tenants' sessions, costs, knowledge base, filesystem;
  the control-plane host itself.
- **Boundary**: the RPC router at `/api/rpc` + SSE at `/api/events/stream`
  + WS at `/api/terminal`. Auth is enforced per-request in
  `packages/core/hosted/web.ts`. Handlers receive a tenant-scoped
  `AppContext` view (`app.forTenant(tenantId)`) so repository lookups are
  transparently filtered; handlers that bypass `app.*` or pass through a
  caller-supplied `tenantId` break this property.

### `packages/arkd` (universal per-compute daemon)

- **Attacker**: a client on the host network that reaches the arkd port
  (:19300), or a compromised agent on the same compute that can speak to
  arkd over loopback.
- **Asset**: arbitrary process/filesystem access on the compute the arkd
  runs on.
- **Boundary**: HTTP listener on `0.0.0.0:19300` (by default) protected by
  a bearer token (`ARK_ARKD_TOKEN`) and an optional workspace-root
  confinement for `/file/*` and routed commands. Session names, cwd, and
  other client-supplied strings must be validated before being
  interpolated into shell or filesystem paths.

### `packages/core/auth` (tenant policy / api keys)

- **Attacker**: an authenticated caller in tenant A attempting to mutate or
  read tenant B's data (keys, policies, usage) by guessing ids.
- **Asset**: other tenants' API keys (credential material) and cost /
  policy records.
- **Boundary**: the `ApiKeyManager` is the sole path through which API key
  rows are mutated; the `UsageRecorder` is the sole path for cost rows.
  Both must accept a tenant scope and refuse to touch out-of-tenant rows.

### `packages/core/observability` (cost / usage)

- **Attacker**: any authenticated tenant calling `costs/*` RPCs.
- **Asset**: other tenants' token counts and dollar spend.
- **Boundary**: `UsageRecorder` holds a `tenantId` and must use it for
  every read and write. The RPC handlers must not forward a
  caller-supplied `tenantId` override.

### `packages/core/hosted` (web server / proxy / terminal bridge)

- **Attacker**: any network caller, especially in hosted multi-tenant
  deployments that expose the web server publicly.
- **Asset**: session-level shell access via the terminal bridge; the
  shared legacy token.
- **Boundary**: the single-token path at `?token=...` is a per-deployment
  secret; any comparison must be constant-time so response latency cannot
  be used as an oracle. The terminal-bridge session id must be validated
  before being interpolated into tmux commands.

## Confirmed findings - fixed in this PR

### Critical

- **C1 - Arbitrary file deletion via `tools/delete`** (`packages/server/handlers/tools.ts:30-34`, CWE-22).
  The `claude-skill` branch called `unlinkSync(source)` with a fully
  caller-controlled `source`. Any RPC caller could unlink arbitrary files
  writable by the server user. Fixed by `isSafeClaudeSkillPath` whitelist
  anchored to `~/.claude/skills` and `<cwd>/.claude/skills`.

- **C2 - Cross-tenant cost read/write** (`packages/core/observability/usage.ts:107-214`,
  `packages/server/handlers/metrics.ts:32-101`, CWE-639).
  `UsageRecorder.getSessionCost` had no tenant filter;
  `getSummary`/`getDailyTrend`/`getTotalCost` accepted a caller-supplied
  `tenantId` and defaulted to "across all tenants" when absent. The
  `costs/*` RPC handlers forwarded `params.tenantId` from the request body.
  A caller in tenant A could query any tenant's spend. Fixed by always
  filtering on the recorder's configured tenant and stripping the
  user-supplied `tenantId` in the handlers. `record()` now also refuses to
  attribute rows to a different tenant.

- **C3 - Privileged host commands exposed to viewers & hosted tenants**
  (`packages/server/handlers/metrics.ts:87-127`, `packages/core/hosted/web.ts`, CWE-284).
  `compute/kill-process`, `compute/docker-logs`, `compute/docker-action`
  were not in `WRITE_METHODS`, so viewers and `--read-only` web clients
  could invoke them. `compute/kill-process` took any pid and signaled it.
  Fixed by adding a hosted-mode refusal, strict pid/container-name
  validation, and adding the mutating variants to `WRITE_METHODS`.

- **C4 - Global cache wipe via `history/rebuild-fts`**
  (`packages/server/handlers/history.ts:41-49`, CWE-284).
  Called `DELETE FROM claude_sessions_cache` / `DELETE FROM
  transcript_index` with no guard. These tables index the local user's
  `~/.claude` transcripts and have no tenant column, so in hosted mode any
  authenticated tenant could wipe the shared cache. Fixed by refusing the
  call in hosted mode.

### High

- **H1 - Non-constant-time bearer token compare (arkd)**
  (`packages/arkd/server.ts:226-232`, CWE-208). Token compared with `===`,
  leaking a per-byte timing oracle. Fixed with `crypto.timingSafeEqual`
  and a length-normalized compare that still runs when lengths differ.

- **H2 - Non-constant-time shared-token compare (web / web-proxy)**
  (`packages/core/hosted/web.ts:250-255`,
  `packages/core/hosted/web-proxy.ts:52-57`, CWE-208). Legacy shared-token
  check used `!==`. Fixed with `timingSafeEqual` plus length normalization.

- **H3 - Session-name injection in arkd `agent/launch`**
  (`packages/arkd/server.ts:640-666`, CWE-78 + CWE-22). The launcher wrote
  to `/tmp/arkd-launcher-${req.sessionName}.sh` and passed `bash <path>`
  as the tmux command without validating `sessionName`. A caller could
  traverse the filesystem and bleed shell metacharacters into the
  executed command (tmux parses its final argv as a shell command). Fixed
  by rejecting `sessionName` that doesn't match `[A-Za-z0-9_-]{1,64}` and
  reapplying the guard to `agentKill`, `agentStatus`, `agentCapture`,
  `isTmuxRunning`.

- **H4 - Cross-tenant api_keys revoke/rotate**
  (`packages/core/auth/api-keys.ts:122-137`, CWE-639). `revoke(id)` and
  `rotate(id)` selected / deleted api_keys rows with no tenant scope. A
  caller that knew or guessed an id could destroy (revoke) or hijack
  (rotate) another tenant's key. Fixed by adding an optional `tenantId`
  parameter that restricts the SQL to that tenant, with new regression
  tests.

### Medium

- **M1 - Hosted-mode filesystem exposure via knowledge handlers**
  (`packages/server/handlers/knowledge.ts:51-75`, CWE-22).
  `knowledge/index`, `knowledge/export`, `knowledge/import` accepted
  arbitrary server-side paths and performed filesystem reads/writes.
  Fine in local single-user mode but a cross-tenant FS surface in hosted
  mode. Fixed by refusing the call in hosted mode.

- **M2 - Hosted-mode filesystem exposure via web handlers**
  (`packages/server/handlers/web.ts`, CWE-22). `mcp/attach-by-dir`,
  `mcp/detach-by-dir`, `repo-map/get`, `knowledge/ingest` all accept
  arbitrary directory paths. Refused in hosted mode.

- **M3 - Transitive dev-only dependency CVEs** (`bun.lock`). `yaml` via
  `vite`, `brace-expansion` via `eslint`/`@typescript-eslint`. Dev-only;
  not shipped in the runtime binary. Recorded here; the fix requires a
  breaking `vite` major bump that is intentionally scoped out of this
  security PR.

## Triaged-as-safe

- **Dynamic column names in repository SQL**
  (`packages/core/repositories/session.ts:220,266`,
  `packages/core/repositories/compute.ts:171`,
  `packages/core/repositories/compute-template.ts:84`,
  `packages/core/knowledge/store.ts:88`). These use whitelist sets
  (`SESSION_COLUMNS`, `COMPUTE_COLUMNS`, plus hard-coded `sets` arrays)
  to guard the template-string SQL. No user-controlled column name
  reaches the query.

- **`packages/core/auth/tenant-policy.ts:88` `ALTER TABLE ... ADD COLUMN`
  via template string**. The column names and types are hard-coded in the
  migration list, not user input.

- **`packages/core/theme.ts:47` `execSync("defaults read ...")`**. Fixed
  string, no user input.

- **`packages/core/hosted/terminal-bridge.ts`**. Session names already
  validated against `^[a-zA-Z0-9_-]+$` before interpolation into tmux
  commands.

- **`packages/core/hosted/web.ts` GitHub webhook signature check**.
  Already uses `crypto.timingSafeEqual`.

- **`packages/core/conductor/conductor.ts` tenant resolution**. Already
  validates Bearer tokens against `api_keys` and refuses to trust
  `X-Ark-Tenant-Id` without a matching key.

- **`packages/router/providers.ts:466,485` Google provider embeds
  `api_key` in URL**. This is the Google AI Studio contract; the URL is
  not logged by arkd or the router. Documented for awareness.

- **`Math.random` usage in tests and `firecracker/compute.ts:455`**. None
  of the hits are used for authentication or secret material; they are
  VM-name suffixes and test fixture names.

- **`claude_sessions_cache` / `transcript_index` tables without
  `tenant_id`**. These are the local-user transcript cache and are
  inherently single-user. The destructive `history/rebuild-fts` endpoint
  now refuses to run in hosted mode rather than retrofit a tenant column.
