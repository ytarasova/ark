# Test coverage + TDD-hygiene audit -- 2026-04-19

Audit of test coverage and TDD hygiene across every package in Ark. Produced
by running `bun test --coverage --concurrency 4` per package and a manual
review of test files for smells.

- `packages/core`
- `packages/compute`

- `packages/router`

## Test smells (file:line references)

### Hardcoded ports (CLAUDE.md violation)

These tests open server sockets on hardcoded ports. CLAUDE.md says: "Never
hardcode a port in a new test. Call `allocatePort()` from
`packages/core/config/port-allocator.ts`, or get it via `app.config.ports.*`
when you have an AppContext." Each occurrence makes the test worker-unsafe
under `--concurrency 4`.

- `packages/core/__tests__/web.test.ts:46` (and 30 more calls 18420..18550)
- `packages/router/__tests__/engine.test.ts:15` (`port: 8430`)
- `packages/router/__tests__/router-usage.test.ts:30` (`port: 18431`)
- `packages/arkd/__tests__/server.test.ts:558` (`port: 19381`)
- `packages/cli/__tests__/daemon.test.ts:74` (`port: 19300`)
- `packages/core/router/__tests__/tensorzero.test.ts:12,18,25`
- `packages/core/router/__tests__/tensorzero-config.test.ts:18,88`
- `packages/compute/__tests__/arkd-backed.test.ts:183`
- `packages/compute/__tests__/local-arkd.test.ts:240,270,382`
- `packages/compute/__tests__/reverse-tunnel.test.ts:10,20-21` (data
  fixtures for probe ports -- not sockets, fine to leave)

The fixture-shaped ones in `arc-json.test.ts`, `docker-provider.test.ts`,
`local-provider.test.ts`, `e2e-compute.test.ts` are payload data (`{ port:
3000 }` in an `arc.json` blob), not socket binds -- they are fine and not
listed here.

### Live network calls inside unit tests

- `packages/core/__tests__/bridge.test.ts` hits Slack + Discord webhook
  URLs during the run; the log shows `slack send failed: Unable to
  connect`. These should be skipped in CI or stubbed with a local HTTP
  fixture.

### Shared git worktrees + `ambiguous argument 'origin/main..HEAD'`

Many core tests spawn real `git init`s and check diff against
`origin/main`, which fails silently when the temp repo has no remote.
Seen repeatedly in `core.log`. Not incorrect per se, but tests that depend
on live git state are fragile. Consider centralizing the test-git setup
helper to seed a dummy origin.

### Oversized single-file suite (smell of test organization)

- `packages/core/__tests__/web.test.ts` -- 331 lines, 31 test cases,
  every one binds a unique hardcoded port. This is the biggest port-smell
  offender in the repo and should be rewritten against
  `app.config.ports.web` + `allocatePort()`.

### Minor

- `packages/core/__tests__/di-container.test.ts:203-231` reaches into the
  legacy `forTest()` API. CLAUDE.md notes this is kept for historical
  compatibility; these tests are acceptable as they validate the legacy
  path, but they should stay the only consumers.

### Not observed

- No tests with `expect()` missing (all `.test.ts` files use `expect`).
- No `vi.fn`/`jest.fn` mocking (package-wide the test style prefers real
  AppContexts plus real routers, which avoids the "testing the mock"
  trap).

## Missing integration tests for critical flows

Already covered reasonably:

- Session lifecycle end-to-end: `packages/core/__tests__/e2e-session-lifecycle.test.ts`
- Auth/tenant boundaries: `packages/core/__tests__/tenant-scoping.test.ts`,
  `tenant-policy.test.ts`, `auth-middleware.test.ts`
- Knowledge-graph indexing: `packages/core/knowledge/__tests__/*.test.ts`

Gaps:

- Compute provision -> dispatch -> teardown: the `ComputePool` +
  `LocalFirecrackerPool` wiring from commits eda02638 and ef293b95 has
  unit tests (`local-firecracker-pool` at 96% lines) but no end-to-end
  test that provisions via the pool, dispatches a session, and tears
  down. Recommend an integration test gated on local firecracker
  availability (use `isFirecrackerAvailable()` and skip otherwise).
- SSE event fan-out: `packages/core/__tests__/sse-bus.test.ts` covers
  the bus; the full web RPC -> SSE round-trip is only exercised in
  `web.test.ts`, which is currently broken (hardcoded ports). Once
  web.test.ts is fixed, an SSE smoke case should be added.
- `packages/compute/providers/remote-arkd.ts` is 18% funcs / 15% lines.
  Given the size of the remote compute surface, this is the riskiest
  single production file without tests (565 lines).

## Reproduce

```bash
mkdir -p /tmp/ark-cov
for p in core compute server router arkd protocol; do
  bun test packages/$p --coverage --concurrency 4 > /tmp/ark-cov/$p.log 2>&1
  grep -E "^All files" /tmp/ark-cov/$p.log | tail -1
done
```
