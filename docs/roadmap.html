Roadmap

What's built, what's in flight, what's next.

Last updated: 2026-04-11. Tests: 2630 pass, 0 fail, 0 lint errors, 0 process leaks.

Recent completions

Items that shipped in the last cycle and are not yet reflected in the status tables below:

  • TensorZero gateway integrated (lifecycle manager, sidecar/native/Docker modes, cost feed-back).
  • LLM Router wired into executor dispatch flow (ANTHROPIC_BASE_URL / OPENAI_BASE_URL injection).
  • Axon replaced by ops-codegraph (@optave/codegraph, TypeScript + Rust, 33 languages, no Python dependency).
  • Codebase auto-index on dispatch -- local honors config, remote always indexes via arkd /codegraph/index.
  • Aider runtime dropped. Active runtimes: claude, claude-max, codex, gemini, goose.
  • Polymorphic transcript parsers (Claude/Codex/Gemini) via TranscriptParserRegistry and DI.
  • Workdir-based session identification (no "latest by mtime" hacks).
  • cost_mode column (api/subscription/free) in usage_records.
  • Compute templates (CLI + Web + RPC, tenant-scoped).
  • Full multi-tenant entity scoping: user_id on sessions, tenant_id on groups/schedules/pools.
  • DB-backed resource stores for control plane mode (DbResourceStore, resource_definitions table).
  • Tenant integration policies (router_required, auto_index_required, router_policy, tensorzero_enabled).
  • Token-budgeted context injection (~2000 tokens max) from the knowledge graph.

Key decisions

  • Temporal for control plane workflow engine. Start with a local event-sourced engine, plan for Temporal hosted backend. Same WorkflowEngine interface.
  • TensorZero replaces hand-rolled LLM provider adapters. Ark keeps routing intelligence (classifier, policies, sticky sessions, cost attribution). TensorZero handles API dispatch, retries, streaming, feedback optimization. Rust sidecar. Apache 2.0.
  • ops-codegraph replaces Axon as the indexer. TypeScript + Rust, 33 languages, no Python dependency.
  • Async Postgres repos are required for hosted scale. Camp 9 priority.
  • Task board: Tasks sit above sessions. Creating a task does not dispatch -- dispatching a task creates a session.
