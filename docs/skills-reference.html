Skills Reference

Reusable prompt fragments that extend agent capabilities.

Overview

Skills are reusable prompt fragments that get injected into an agent's system prompt at dispatch time. They let you encode domain knowledge, coding standards, review criteria, or any recurring instructions once and attach them to any agent.

Skills are YAML files with a prompt field containing the text to inject. Attach skills to agents via the skills field in agent YAML.

Three-Tier Resolution

Skills are resolved using a three-tier lookup (highest priority first):

TierLocationScope
Project.ark/skills/<name>.yamlCommitted to repo, shared with team
Global~/.ark/skills/<name>.yamlUser-wide, personal preferences
Builtinskills/<name>.yamlShipped with Ark

A project-level skill with the same name as a builtin skill overrides it. This lets you customize builtin behavior per-repo without modifying Ark itself.

Skill YAML Schema

name: my-skill
description: "What this skill teaches the agent"
prompt: |
  Multi-line prompt text that gets injected
  into the agent's system prompt at dispatch.
tags: [review, quality]
FieldTypeRequiredDescription
namestringYesUnique skill identifier (used in agent YAML skills field)
descriptionstringYesBrief description shown in ark skill list
promptstringYesPrompt text injected into agent's system prompt
tagsstring[]NoCategorical tags for organization

Builtin Skills

code-review

Reviews code changes for bugs, security issues, performance problems, and style violations. Rates each issue P0-P3 with specific fix suggestions.

name: code-review
description: Reviews code changes for bugs, style, and best practices
tags: [review, quality]

Prompt behavior: Instructs the agent to run git diff main...HEAD, check for bugs/security/performance/style, and produce structured output with P0-P3 severity ratings.

Used by: The reviewer agent in the default and pr-review flows.

test-writing

Writes comprehensive tests for existing code, covering happy paths, edge cases, error conditions, and boundary values.

name: test-writing
description: Writes comprehensive tests for existing code
tags: [testing, quality]

Prompt behavior: Analyzes specified code and writes tests following the project's existing test patterns and framework conventions.

plan-audit

Audits implementation plans against spec requirements. Produces a coverage matrix and gap analysis.

sanity-gate

Pre-implementation sanity check. Validates that the task is well-defined and feasible before execution begins.

security-scan

Scans code changes for security vulnerabilities, secrets exposure, and unsafe patterns.

self-review

Agent self-reviews its own changes before reporting completion. Catches obvious issues before human review.

spec-extraction

Extracts a structured specification from tickets, PRs, or issue descriptions.

Attaching Skills to Agents

Add skills to an agent definition via the skills field:

name: my-reviewer
description: "Custom reviewer with code-review skill"
model: opus
skills: [code-review]
system_prompt: |
  Review the changes on branch {branch} in {repo}.

At dispatch, Ark resolves each skill name, reads the prompt text, and appends it to the agent's system prompt. Multiple skills are concatenated in order.

Creating Custom Skills

From the CLI

$ ark skill create my-lint --prompt "Run eslint with --fix before committing" -d "Auto-lint skill"
$ ark skill create my-lint --scope project   # save to .ark/skills/ instead of ~/.ark/skills/

From a YAML file

$ ark skill create --from my-skill.yaml
$ ark skill create --from my-skill.yaml --scope project

Manually

Create a YAML file directly in the appropriate directory:

# ~/.ark/skills/my-style.yaml (global)
name: my-style
description: "Enforce our team's coding style"
prompt: |
  Follow these style rules:
  - Use 2-space indentation
  - Prefer const over let
  - No default exports
tags: [style, linting]

Skill Extraction

Ark can analyze completed agent conversations and extract reusable procedures as skills. Extracted skills are named extracted-<sessionId>-N and saved to the global tier. Review and rename them before attaching to agents.

CLI Commands

CommandDescription
ark skill listList all skills across all tiers (shows source tier)
ark skill show <name>Show skill details and full prompt text
ark skill create <name>Create a new skill (--prompt, --from, --scope)
ark skill delete <name>Delete a global or project skill (cannot delete builtins)

Create options

OptionDescription
-p, --prompt <text>Skill prompt text (required unless --from)
-d, --description <desc>Skill description
--from <file>Create from a YAML file
-s, --scope <scope>global (default) or project
--tags <tags>Comma-separated tags

Delete options

OptionDescription
-s, --scope <scope>global (default) or project
Note: Builtin skills cannot be deleted. To override a builtin, create a project or global skill with the same name -- the higher-priority tier wins.

Skills in the TUI

Skills appear in the Tools tab (7) under the "Ark Skills" category. You can browse, view, and use skills from the TUI without the CLI.
