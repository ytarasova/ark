Recipes Reference

Session templates for quick-launching common workflows.

Overview

Recipes are session templates with variables, default flows, and optional agent overrides. They let you define reusable session configurations -- instead of typing --repo . --flow bare --summary "..." every time, create a recipe and launch with --recipe quick-fix.

Three-Tier Resolution

Recipes are resolved using the same three-tier lookup as skills and agents:

TierLocationScope
Project.ark/recipes/<name>.yamlCommitted to repo, shared with team
Global~/.ark/recipes/<name>.yamlUser-wide personal templates
Builtinrecipes/<name>.yamlShipped with Ark

Recipe YAML Schema

name: my-recipe
description: "What this recipe does"
flow: bare                   # flow to use
agent: worker                 # optional: override default agent
variables:
  - name: repo
    description: "Repository path"
    required: true
  - name: summary
    description: "Task description"
    required: true
  - name: ticket
    description: "Ticket ID (optional)"
defaults:
  summary: "Default summary text"
agent_overrides:              # optional: per-stage agent tweaks
  implement:
    system_prompt_append: |
      Extra instructions for the implement stage.
config:
  auto_merge: false
  breakpoint: review
FieldTypeRequiredDescription
namestringYesUnique recipe identifier
descriptionstringYesShown in ark recipe list
flowstringNoFlow to use (defaults to repo/global default)
agentstringNoOverride the default agent for the flow
variablesVariable[]NoPrompted inputs when using the recipe
defaultsobjectNoDefault values for session fields
agent_overridesobjectNoPer-stage agent configuration overrides
configobjectNoSession config (auto_merge, breakpoint, etc.)

Variable Schema

FieldTypeDescription
namestringVariable name (used in templates as {name})
descriptionstringPrompt text shown when asking for input
requiredbooleanWhether the variable must be provided
defaultstringDefault value if not provided
typestringVariable type hint (e.g. string)

Builtin Recipes

quick-fix

Quick single-agent bug fix using the bare flow with a worker agent. No planning or review stages -- just fix and done.

FieldValue
Flowbare
Agentworker
Variablesrepo (required), summary (required)
 
$ ark session start --recipe quick-fix --repo . --summary "Fix null check in auth" --dispatch

feature-build

Full plan-implement-review cycle using the default flow. Best for new features that benefit from a planning stage before implementation.

FieldValue
Flowdefault
Variablesrepo (required), summary (required), ticket (optional)
 
$ ark session start --recipe feature-build --repo . --summary "Add OAuth support" --dispatch

code-review

Review current branch changes using the reviewer agent in a bare flow.

FieldValue
Flowbare
Agentreviewer
Variablesrepo (required)
Default summary"Review the code changes on this branch"
 
$ ark session start --recipe code-review --repo . --dispatch

fix-bug

Bug fix recipe using the default flow with agent overrides that instruct the implementer to reproduce first, write a failing test, then fix with minimal changes.

FieldValue
Flowdefault
Variablesticket (required), repo (required)
Configauto_merge: false, breakpoint: review

new-feature

New feature recipe using the default flow with no special overrides. Similar to feature-build but uses the older parameter syntax.

FieldValue
Flowdefault
Variablesticket (required), repo (required)
Configauto_merge: false

ideate

Brainstorming recipe for exploring ideas before implementation.

islc

Full ISLC (Intent-to-Software Lifecycle) recipe using the islc flow with 7-stage pipeline.

islc-quick

Shortened ISLC recipe for quick implementations with verification.

Using Recipes

Pass --recipe to ark session start:

$ ark session start --recipe quick-fix --repo . --summary "Fix typo" --dispatch
$ ark session start --recipe feature-build --repo . --summary "Add caching" -t PROJ-42 --dispatch

Variables defined in the recipe are prompted interactively if not provided via CLI flags. The --dispatch flag dispatches the session immediately after creation.

Creating Custom Recipes

From a YAML file

$ ark recipe create --from my-recipe.yaml
$ ark recipe create --from my-recipe.yaml --scope project

From an existing session

Capture a successful session's configuration as a reusable recipe:

$ ark recipe create --from-session s-abc123 --name my-template
$ ark recipe create --from-session s-abc123 --name my-template --scope project

This extracts the session's flow, agent, repo, summary, and config into a recipe YAML file.

Manually

Create a YAML file directly in ~/.ark/recipes/ (global) or .ark/recipes/ (project).

Deleting Recipes

$ ark recipe delete my-recipe
$ ark recipe delete my-recipe --scope project
Note: Builtin recipes cannot be deleted. To override a builtin, create a project or global recipe with the same name.

CLI Commands

CommandDescription
ark recipe listList all recipes across all tiers
ark recipe show <name>Show recipe details, variables, and defaults
ark recipe createCreate from YAML (--from) or session (--from-session)
ark recipe delete <name>Delete a global or project recipe

Create options

OptionDescription
--from <file>Create from a YAML file
--from-session <id>Create from an existing session
-n, --name <name>Recipe name (required with --from-session)
-s, --scope <scope>global (default) or project

Recipes in the TUI

Recipes appear in the Tools tab (7) under the "Recipes" category. You can also use a recipe when creating a new session (n) in the Sessions tab by selecting a recipe from the form.

Recipe Evaluation

Test a recipe by running it multiple times and collecting results:

$ ark eval quick-fix -n 3

Reports success rate, average duration, and average cost across iterations. Useful for benchmarking different recipe configurations.
