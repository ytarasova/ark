# Agent 8 ŌĆö Accessibility Audit

## Summary

The web dashboard ships decent icon-only labeling and at least one sr-only h1, but has significant gaps: no focus traps in any modal/drawer/command-palette, no focus-return on close, a global single-key navigation handler that fires when non-input elements have focus, missing `role="tablist"` wrapper around `ContentTabs`, a purely visual toast (no `aria-live`), no `prefers-reduced-motion` handling anywhere, and contrast failures for muted/faint text plus the primary-button foreground on several themes. Most pages lack a top-level `<main>` landmark and an h1; only `SessionsPage` and `LoginPage` have one. E2e has zero a11y assertions.

## Top 5 A11y Bets

1. **Add focus trap + focus restoration + dialog roles to every overlay** (A4 + B6 + B7). Every sighted keyboard user and SR user is affected. Low cost with Radix `Dialog` replacement for the home-rolled `Modal`/`DetailDrawer`/`CommandPalette`.
2. **Make Toast announce with `role="alert"`/`role="status"` + `aria-live`** (B2). Right now error toasts ŌĆö the highest-information surface ŌĆö are invisible to SR users after a dispatch failure.
3. **Fix primary-button + fg-faint contrast across three themes** (C1, C2, C3). The warm-obsidian light primary fails at 2.7:1; fg-faint helper text fails across every theme.
4. **Respect `prefers-reduced-motion`** (E1, E2). A global media query in `styles.css` disables pulse/shimmer/slide with one block; vestibular users get instant relief.
5. **Fix focus-indicator regressions** (D1, D5, D7). Several core inputs (command palette, session list search, new session textarea) have `outline-none` without a replacement; trivial to fix.

## Recommended A11y E2e Invariants

1. **Overlay focus discipline**: open `NewSessionModal`/`CommandPalette`/`DetailDrawer`, press Tab 10├Ś ŌĆö active element must stay within the overlay. Close it ŌĆö `document.activeElement` must equal the trigger button. Guards against A4.
2. **Keyboard-only navigation**: with Tab alone, reach and activate each IconRail item; pressing Enter on the Sessions rail button navigates to sessions; `aria-current="page"` updates. Guards against A1/B12 regressions and ensures nav stays keyboard-accessible.
3. **SR announcements**: after triggering a toast error (e.g. dispatch with daemon offline), assert a node with `role="alert"` contains the message; after opening a dialog, assert `role="dialog"` + `aria-modal="true"` is present. Guards against B2 and B6.

## Electron Notes

- `packages/desktop/preload.js` exposes only `{platform, isElectron}` and adds `is-electron`/`is-macos` body classes ŌĆö safe, no a11y weakening.
- `drag-region` in `IconRail.tsx:68` applies `-webkit-app-region: drag` to the logo container. This **blocks click/keyboard focus on non-macOS** for child interactive controls unless they add `.no-drag`. The logo `div[role="button"]` (`IconRail.tsx:70-83`) lives inside the drag region; confirm it remains keyboard-activatable on Windows/Linux builds (Chromium does allow focus, but clicks are swallowed by drag). Flag as medium; not currently exercised by e2e.
- Window chrome uses `titleBarStyle: "hidden"` on Windows/Linux (`main.js:326`) but provides **no custom minimize/maximize/close controls** in the React UI. Non-macOS Electron users have no visible way to min/max/close via UI ŌĆö they rely on OS shortcuts (Alt+F4, Win+Up). High severity for packaged Electron on Windows/Linux.
