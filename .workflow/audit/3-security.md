# Agent 3 — Security + Tenant Isolation Audit

## Summary

Ark is currently architected for single-tenant local use but ships multi-tenant scaffolding (`tenant_id` columns, per-request `AppContext.forTenant(id)`). The scaffolding is **not safe to enable**: tenant identity is trusted from client-supplied headers in the conductor; the default auth config is `enabled: false`; multiple REST and file-write endpoints bypass tenant scoping entirely. Path-traversal and command-injection surfaces are pervasive throughout `packages/core/services/*` and `packages/compute/providers/ec2/*` — anywhere user-controlled strings (`session.id`, `att.name`, `session.workdir`, `compute.config.ip`, `body.path`) flow into `join()`, `writeFileSync()`, or `sshExec(key, ip, <template>)`. The conductor exposes full session state without auth over localhost; arkd's `/file/write`, `/file/read`, `/exec` endpoints take arbitrary paths from the authenticated caller with zero scoping. `MarkdownContent.tsx` is genuinely safe (regex + JSX). Seven moderate CVEs (Vite, esbuild, yaml, brace-expansion) — all dev-tooling transitives, no prod impact, all have fixes. No e2e deny-path coverage exists for any security surface.

## Top 5 Security Bets

1. **Fix tenant identity (S-02, S-03).** Today any local caller to the conductor can set `X-Ark-Tenant-Id: victim` and read/write other tenants' sessions. Drop header-based tenant derivation; require API-key validation always when auth is enabled; route `/api/sessions` through `appForRequest`. Nothing else in the multi-tenant story is safe until this is.
2. **Sanitize `att.name` + confine attachment writes (S-01).** One-line fix (`basename`) prevents the web UI from writing `../../../.ssh/authorized_keys` into the worktree. Already documented as a known issue; ship the fix.
3. **Shell-escape everywhere in ec2 providers (S-05, S-06, S-07, S-08).** `packages/compute/providers/ec2/shell-escape.ts` exists but is under-used. Wrap every `sshExec(key, ip, \`...\${var}...\`)` in `shellEscape(var)`, or refactor to argv by uploading a script and running `bash script.sh`.
4. **Confine arkd `/file/*` and `/exec` to the session's workdir root (S-04, S-13).** Arkd currently trusts the bearer for full host fs access. A compromised single session token is a full worker-host takeover. Pass `session_id` + compute root; reject paths outside.
5. **Add a `security.spec.ts` to e2e (S-25).** Seven deny-paths worth testing: unauthenticated RPC, wrong-token RPC, `fs/list-dir` with `../`, attachment with traversal name, cross-tenant session read, write method as viewer role, WebSocket terminal without auth. This is cheap ("missing-deny-e2e" is always a write-once investment) and prevents future regressions as new handlers land.
