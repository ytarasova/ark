# Ark

Autonomous agent ecosystem. Orchestrates AI coding agents through DAG-based SDLC flows with 11 compute providers, knowledge graph, LLM router, and multi-tenant control plane. Supports Claude Code, Codex, Gemini CLI, and Goose runtimes. Bun + tmux only.

## Commands

```bash
make install          # bun install + symlink ark to /usr/local/bin
make test             # run all tests (parallel)
make test-file F=path # run a single test file
make lint             # ESLint (zero warnings allowed)
make format           # Prettier auto-fix
make dev              # hot-reload: API (:8420) + Vite HMR (:5173) + auto-starts daemon
```

## Before Committing

```bash
make format && make lint   # MUST pass; CI rejects otherwise
```

## Key Rules

- **Bun-only** -- uses `bun:sqlite`, `Bun.serve()`, `Bun.sleep()`
- **ES module `.js` extensions required** -- `import { foo } from "./bar.js"`
- **`strict: false`** in tsconfig; implicit `any` allowed
- **Tmux required** -- sessions launch agents in tmux (`ark-s-<id>`)
- **Never use em dashes** (U+2014) -- use hyphens or double dashes
- **Never hardcode ports** -- use `app.config.ports.*` or `allocatePort()`
- **Nunjucks templating** -- `{{var}}` double braces in YAML definitions

## Testing

```bash
make test                                                # full suite
make test-file F=packages/core/__tests__/session.test.ts # single file
```

Use `AppContext.forTestAsync()` for test isolation (fresh arkDir + ephemeral ports).

## Structure

```
packages/{cli,core,compute,arkd,router,conductor,protocol,web,desktop,types,workspace}
agents/       -- Agent YAML definitions
runtimes/     -- Runtime definitions (claude, codex, gemini, goose)
flows/        -- Flow definitions (autonomous-sdlc, quick, fan-out, etc.)
skills/       -- Builtin skills
```

Key entry points: `AppContext` (app.ts), `SessionService` (services/session.ts), `session-orchestration.ts`.
