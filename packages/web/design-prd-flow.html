PG Call-outcome capture SDK
What it does
Embeds the approved telephony SDK (Twilio/Exotel) to listen for call-end events and outcome codes. Bridges native call lifecycle to a structured event the backend can consume.
User story
As an FSE, I want the app to automatically record who I called and whether the call was answered, so I spend 5 seconds logging per call vs the next task.
Acceptance criteria
Given a call initiated from the GQ App, when the call ends, then the SDK emits a CALL_ENDED event with contact ID, duration, and disposition code.
Given a CALL_ENDED event, when the app receives it, then a pre-populated call log is created with contact name, timestamp, duration, and outcome.
Given the device is offline, when a call ends, then the event is queued in local storage and synced within 30s of reconnection.
Given the SDK fails to initialize, when the app starts, then the FSE sees a non-blocking warning and can still make calls via the native dialer (graceful degradation).
